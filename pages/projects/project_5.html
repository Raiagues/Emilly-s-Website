Project Details

Description

This project focuses on fault injection in memory systems to evaluate fault tolerance and protect against side-channel attacks. The project involves injecting Single Event Upset (SEU) faults, setting up a Design Under Test (DUT) with a testbench for fault simulation, and automating the process using ModelSim and TCL scripts. Results are analyzed to identify critical vulnerabilities, and strategies are proposed to enhance fault tolerance.

  • Context: Project for the course "Sécurité et sûreté matérielle" in the Embedded Systems and Connected Objects master's program.
  • Local: Institut Polytechnique de Grenoble (INP Grenoble), Grenoble, France
  • Duration: 2 months
  • Team Size: 2 people
  • Timeline: Oct 2024 - Nov 2024

Technologies Used

  • Programming Languages: TCL
  • Tools: ModelSim
  • Concepts: Fault Injection, Fault Tolerance, Attack Prevention

Tasks

  • Injecting SEU faults to evaluate fault tolerance and prevent potential attacks.
  • Setting up a Design Under Test (DUT) and creating a testbench to simulate faults.
  • Automating simulations with ModelSim and TCL scripts to streamline fault injection.
  • Analyzing results to identify critical components and vulnerabilities, and proposing fault-tolerance techniques.
